# V2.0.0 Release Notes

V1.12.5 is the final local stable release in the 1.x line. V2.0.0 introduces an MVP for multi-turn context inside the current browser conversation.

## Conversation behavior

- Each conversation has an isolated ID and compact user/assistant messages. Successful Mock, Real, fallback, and clarification responses are persisted; failed requests are not.
- Refresh restores the active conversation. “New conversation” creates a new ID and empty context without deleting Feedback, Evaluation, RAG Test History, or legacy Chat Run History.
- The context window keeps at most 6 rounds, 12 messages, and 6,000 historical characters; individual messages are limited to 2,000 characters and older complete messages are dropped first.
- Legacy flat Chat Run History is copied into a default conversation on first use while its original storage key and report behavior remain unchanged.

## Agent and privacy boundaries

- `{ question: "..." }` remains a valid API request. Optional `conversationContext` is filtered on both client and server.
- Router, RAG, and tool parameters continue to use the current question. Bounded history is used for follow-up resolution and Real generation only.
- Real prompts order system rules before explicitly marked untrusted history and the current question. Historical prompt-injection text cannot become a system instruction.
- Mock mode deterministically resolves “订单10001可以退货吗？” followed by “那需要准备什么材料？”. A newly created conversation does not inherit that context.
- Ops JSONL never stores conversation messages or context payloads. It records only whether context was applied, the bounded message count, and whether truncation occurred.

This MVP does not include long-term memory, profiles, automatic summaries, server-side conversation persistence, databases, multi-device sync, or a full conversation sidebar. Real API was not invoked locally and requires a controlled post-deployment smoke test.
