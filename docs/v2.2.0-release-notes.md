# V2.2.0 Release Notes — 服务端存储与持久化

## 版本目标

V2.2.0 将会话、消息、滚动摘要和用户知识文档从“只能依赖当前浏览器 localStorage”升级为可选的 PostgreSQL 服务端持久化，同时保留 V2.0.4 / V2.1.0 本地数据兼容和明确的故障模式。

本版本使用 Prisma 6.19.3，正式数据库目标为 PostgreSQL。数据库未启用时应用继续使用 localStorage；数据库已启用且健康时服务端成为数据源；数据库已配置但不可用时进入 degraded 模式，写操作不会静默降级为本地成功。

## 新增能力

- **PostgreSQL Schema**：增加 Workspace、Conversation、Message、KnowledgeDocument、KnowledgeChunk、ImportJob 和 StorageMigration。
- **匿名工作区隔离**：服务端签发签名的不透明 HttpOnly Cookie；生产环境使用 Secure，SameSite 至少为 Lax。数据库仅保存会话令牌哈希，所有业务访问均限定服务端解析的 workspaceId。
- **Conversation Repository**：Local / Server 适配器共用 list、get、create、rename、delete、appendTurn、Regenerate 和 Edit & Resend 语义。
- **Knowledge Repository**：Local / Server 适配器共用知识文档 CRUD、启用状态和 chunks 读取；服务端 RAG 读取当前工作区已启用文档与分块。
- **事务与 CAS**：完整 turn、Regenerate、Edit & Resend、Summary Patch 和 revision 在同一事务中提交；CAS 冲突返回 409，不留下部分写入。
- **滚动摘要持久化**：Conversation Summary 作为可选原子 JSON 保存，继续兼容 text、throughMessageId、updatedAt、version: 1 和 sourceMessageCount。
- **localStorage 迁移**：用户先明确允许上传本地正文，服务端预检后再进行第二次执行确认；幂等导入返回冲突统计和迁移状态，原 localStorage key 与数据不删除。
- **迁移一致性**：JSON/JSONB 使用规范化 checksum；同一 migrationId 只能对应同一数据包。并发重复提交通过唯一约束、有限结果复核与数据包指纹校验返回同一幂等结果；异数据包或未能复核的并发竞态返回 409，不会静默重复导入。
- **知识备份恢复**：服务端以单个 Serializable 事务替换当前工作区备份并重建 chunks，失败时不留下部分恢复状态。
- **三种存储模式**：local、server、degraded 均有显式状态和中文提示。
- **请求体与缓存边界**：存储写路由在完整缓冲前按字节与字符上限读取请求体，超限即取消读取并返回安全错误。工作区业务响应与 Ops 使用私有 `no-store` 缓存策略并按 Cookie 隔离缓存变体；公共存储状态响应同样禁止缓存。
- **Agent 同源边界**：普通与流式 Agent POST 在解析请求或创建匿名工作区前执行同源校验，跨站请求不会触发 Workspace 写入、RAG 或 Agent Pipeline。
- **只读工作区安全**：无有效匿名会话的只读 GET 不签发 Cookie、不创建空 Workspace；只有需要持久化的写请求才会在必要时建立工作区。
- **安全状态与 Ops**：只暴露存储模式、配置/健康状态和聚合计数，不暴露连接串、Cookie、Secret 或业务正文。

## 兼容性

- V2.0.4 和 V2.1.0 Conversation 缺少 revision 或 Summary 时可由现有 sanitizer 安全读取。
- 原 Conversation、Knowledge 和 Client Storage key 不改变。
- Conversation 上限 10 个、单会话消息上限 100 条和现有知识文档容量限制保持不变。
- Legacy Chat History 迁移入口保留。
- localStorage 迁移保留原 ID、时间、消息顺序、runId、安全 assistant details 和合法 Summary cursor。
- 服务器已有相同 ID 时不会静默覆盖；相同内容计为 skipped，不同内容计为 conflicted。

## Conversation Context Manager 保持不变

- Router、RAG、Tool、History Selector、Rolling Summary 和 Context Manager 仍生成唯一 Context Plan。
- Mock / Real、Streaming / Non-streaming 共用同一 Context Plan 与 Summary Patch。
- Summary 不包含当前问题或本次 assistant answer。
- 没有增加第二次模型摘要调用。
- Summary 正文不进入 Ops、Context Trace、assistant details 或默认日志。

## 安全边界

- 匿名 Cookie 只提供浏览器级工作区隔离，不是完整登录、多租户认证、账号恢复或企业组织权限系统。
- 不接入对象存储、Knowledge Pack 批量导入、embedding、向量数据库、长期记忆、用户画像或跨会话记忆。
- 不保存 pending、streaming partial、aborted 或失败回答。
- 不接受客户端传入的 workspaceId 作为授权依据。
- 不提交真实数据库连接串、密码、API Key、Token 或 Secret。

## 数据库与运行脚本

```bash
npm run db:generate
npm run db:validate
npm run db:migrate:dev
npm run db:migrate:deploy
npm run db:status
npm run test:storage
npm run test:storage:postgres
```

生产或共享环境只应执行经过审查的 `db:migrate:deploy`，不得运行 destructive reset。

## 发布与部署状态

本文记录 V2.2.0 Git 发布范围。最终测试数字和 Git commit/tag 指向以发布任务的最终报告为准，不在验证完成前预填。

V2.2.0 Git 发布不包含腾讯云应用部署、服务器 Release、Canary 或 PM2 切换。腾讯云线上应用继续保持 V2.0.4，后续部署需要单独授权与独立的数据库迁移/回滚计划。
