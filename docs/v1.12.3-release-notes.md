# V1.12.3 Release Notes

## Agent Workspace Structure Refactor

V1.12.3 separates the Chat workspace by responsibility without changing the public `AgentWorkspace` entry point, Agent API protocol, Client Storage keys, or existing product behavior.

- `useAgentWorkspace` centralizes URL question prefill, LLM status and health checks, the single Agent request path, response state, clarification and rate-limit handling, feedback runId submission, and unmount-safe async state updates.
- `AgentFeedbackPanel` renders feedback controls from explicit props and does not access browser storage or call Agent APIs.
- `AgentWorkspace` remains the composition entry and retains answer, evidence, tool, Trace, clarification, fallback, and Chat History presentation.

## Validation

- Unit tests continue to cover the existing API, storage, privacy, feedback, and Agent regression behavior.
- Playwright adds Chat Workspace coverage for URL prefill, Mock requests, response rendering, clarification and fallback display with missing optional data, rate-limit messages, feedback runId association, and refresh-safe local history.
- Browser tests use the isolated local E2E app and do not load `.env.local`, call a Real API, or persist test data to the service.

## Scope Boundary

This release does not change the Agent API contract or migrate online data. Real API validation remains the V1.12.1 deployment smoke-test baseline; V1.12.3 does not invoke a Real API.
