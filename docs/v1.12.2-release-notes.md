# V1.12.2 Release Notes

V1.12.2 closes the post-deploy privacy and browser-regression gaps found during the V1.12.1 rollout.

## Ops privacy

- Ops summaries no longer return complete user questions, including short prompts.
- Order numbers, phone numbers, email addresses, identity-style numbers, and long numeric strings are masked before storage and when legacy records are summarized.
- Scenario, intent, response mode, tool, source-count, error, and rate-limit aggregates remain unchanged.

## Browser E2E coverage

- Playwright validates Knowledge Backup export, preview counts, merge restore, replace confirmation, invalid JSON, oversized backups, persistence after refresh, and cleanup.
- Playwright validates legacy-array migration, invalid-record filtering, refresh persistence, and corrupted-JSON recovery for Chat History, Feedback, Evaluation History, and RAG Test History.
- E2E runs from an isolated local application directory and excludes `.env*`, runtime data, and test documents. No real model call or real credential is required.

## Verification baseline

- Unit tests: 18 files, 55 tests.
- Browser E2E: 3 scenarios.
- Full Mock Evaluation: 80/80, 100%.
- Typecheck and production build pass.
- Production dependency audit has no high or critical findings; the known two moderate Next.js-transitive PostCSS findings remain documented.
