# V2.2.1 Release Notes — 企业知识包与导入流程

## 版本目标

V2.2.1 在 V2.2.0 PostgreSQL 服务端知识存储基础上，增加工作区隔离的企业知识包、多文件导入预览、文件安全解析、重复检测、冲突处理、分块质量诊断和可恢复导入任务。

## 新增能力

- **企业知识包**：创建、列表、详情、修改和删除均限定当前 Workspace；默认删除知识包时保留并移出文档。
- **批量导入**：单批支持最多 10 个 TXT、Markdown、文本型 PDF 或 DOCX 文件，客户端与服务端双重校验。
- **导入前预览**：在创建正式知识文档前展示推断标题、字符数、预计分块、受限 chunk preview、质量等级、警告和重复状态。
- **元数据编辑**：确认前可调整标题、标签、来源、启用状态、建议问题和所属企业知识包；metadata 仅接受受限安全 JSON。
- **两级重复检测**：规范化正文 checksum 识别精确重复，并通过标题、文件名和安全元数据识别疑似重复；既检查当前 Workspace 已有文档，也检查同一上传批次。
- **明确冲突策略**：支持跳过、替换现有文档、作为新文档导入；替换复核当前 Workspace 和 revision，不静默覆盖。
- **持久化导入任务**：ImportJob / ImportItem 保存预览、进度和结果；即使浏览器短期任务指针丢失，页面刷新后也可从当前 Workspace 最近 10 个可恢复任务中发现并恢复。
- **幂等与资源边界**：重试复用批次幂等键并在解析前命中；单批解析并发为 2，进程级预览总并发为 4、同 Workspace 为 1，活动任务、临时正文和 24 小时预览保留均有集中上限。
- **并发与失败隔离**：每次 claim 一个项目并使用 lease 防重复消费；单个文档与 chunks 同事务写入，一个文件失败不回滚其他成功文件。
- **部分失败与重试**：汇总成功、失败、跳过和冲突数量；保留提取正文的处理失败项可安全重试，已成功项不会重复导入；解析失败需重新选择原文件。
- **RAG 即时接入**：成功导入且 enabled 的文档和 chunks 立即参与当前 Workspace 的 RAG；删除或替换后不会继续检索旧 chunks。
- **安全 Ops 指标**：增加知识包、任务、项目、成功、失败、冲突、重试、平均耗时和白名单化错误/重复类型分布。

## 文件解析与限制

- TXT / Markdown：严格 UTF-8、BOM 和换行规范化，拒绝编码错误与异常控制字符。
- PDF：使用 `pdfjs-dist 5.6.205` 提取文本，关闭动态代码求值，限制 200 页；无可提取文本时明确失败。
- DOCX：先校验 ZIP entry、解压大小、压缩比、路径、加密/宏/嵌入结构，再使用 `mammoth 1.12.0` 提取；`yauzl 3.4.0` 用于受限结构检查。
- 单文件 5 MiB、单批 25 MiB、提取后 120,000 字符、最多 500 chunks，原始上传文件不长期保存。

## 兼容性

- V2.2.0 的 Workspace、Conversation、Message、KnowledgeDocument、KnowledgeChunk、ImportJob 和 localStorage 数据继续可读。
- `KnowledgeDocument.packId` 继续表示内置 RAG 分类；新增 `knowledgePackId` 表示工作区企业知识包。
- local 模式保留原单文档本地导入和本地 RAG。
- server 模式启用完整知识包与批量导入。
- degraded 模式禁止启动或推进正式导入，不会静默写入 localStorage 假装成功。
- Router、Conversation Context Manager、Rolling Summary、Mock / Real 和 Streaming 语义不变。

## 明确边界

- 不支持 OCR，扫描版/图片型 PDF 不提取文字。
- 原始上传文件不持久保存。
- 未接入腾讯云 COS 或其他对象存储。
- 未增加完整登录、企业组织权限、长期记忆、用户画像或跨会话记忆。
- 未升级 Node 22；Node 基线仍为 20.19.5，Prisma 仍为 6.19.3。
- 没有修改 `.env.local`，没有提交真实数据库连接或 Secret。

## 发布状态

本 Release Notes 描述 V2.2.1 Git 发布范围。最终测试数字、feature/master CI 和 Stable Tag 指向以发布任务最终报告为准。

V2.2.1 Git 发布不包含腾讯云应用部署、生产 PostgreSQL migration、服务器 Release、Canary 或 PM2 切换。腾讯云继续运行 V2.0.4，生产部署需要另行授权。
