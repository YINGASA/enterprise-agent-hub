# V2.2.2 Release Notes — 依赖兼容与生产导入加固

## 版本定位

V2.2.2 是 V2.2.1 企业知识包与导入流程的生产加固版本。它聚焦 Node 20 运行兼容、Real API 传输、导入资源边界、claim/lease 恢复、PostgreSQL 查询索引、故障恢复和发布前检查，不增加大型产品功能或主界面改版。

## 主要变化

- **Node 20 依赖兼容**：将会在代理 Real API 路径实际失败的直接依赖 `undici@8.5.0` 精确固定为 Node 20 兼容的 `undici@7.28.0`，不使用 override 或强制安装。
- **Real API 加固**：非流式请求传播 Request AbortSignal；调用方取消与内部 timeout 分开处理；ProxyAgent 在成功、失败、超时和取消后关闭；本地假服务覆盖 JSON、SSE、代理、401/429/500、异常响应和网络失败。
- **集中资源限制**：批次、文件、提取字符、PDF 页数、DOCX 解压、chunks、解析并发、任务/单项 timeout、最大 retry、lease 和临时正文保留由同一配置模块管理。
- **导入恢复**：增加 claim 续租、过期 lease 重领、最大重试、进程重建恢复和终态幂等；cancelled 与 completed 项不会重新消费。
- **PostgreSQL 查询加固**：为会话列表、runId、规范化文档标题/文件名、任务状态和 claim queue 增加与真实查询匹配的索引；重复检测优先走 checksum 与规范化字段，不再默认加载工作区全部文档正文。
- **安全健康检查**：新增 `/api/health`，只输出应用、数据库、migration、存储模式、Real API 配置、parser、Node 和版本安全状态。
- **生产启动检查**：新增 `npm run production:check`，在服务端存储开启但数据库、Secret、migration 或关键 Schema 未就绪时非零退出。
- **故障与压力门禁**：增加 Node 20 Real API、资源超限、claim/lease、处理器中断、PostgreSQL migration/查询计划、degraded 恢复和受限压力测试。

## 数据与兼容性

- Prisma / Prisma Client 保持 6.19.3，Node 保持 20.19.5。
- V2.2.2 migration 可从 V2.2.1 安全向前部署，不删除旧业务数据。
- V2.0.4、V2.1.0 localStorage key 与数据继续保留；本版不要求清空浏览器数据。
- Conversation Context Manager、Rolling Summary、Streaming、Regenerate、Edit & Resend 和 CAS 业务语义不变。
- server 模式仍以 PostgreSQL 为唯一正式写入来源；degraded 不会伪装成本地写入成功。

## 明确边界

- 不包含腾讯云 COS 或其他对象存储。
- 不包含 OCR，扫描型 PDF 仍不做文字识别。
- 不包含向量数据库、长期记忆、用户画像、账号登录或成员权限。
- 不升级 Node 22/24，不升级 Prisma，不执行 `npm audit fix`。
- 不保存原始上传文件。
- 本次只发布 Git 版本，不部署腾讯云，不执行生产 migration、服务器 Release、Canary 或 PM2 切换；腾讯云继续运行 V2.0.4。
