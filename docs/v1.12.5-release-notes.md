# V1.12.5 Release Notes

Shared TypeScript definitions are split by domain while `@/types` remains the stable explicit type-only public barrel.

- `common`: shared UI and scenario primitives.
- `agent`: routing, pipeline, LLM, API response, and run metadata.
- `knowledge`: documents, retrieval, RAG diagnostics, import, backup-related types.
- `tools`, `feedback`, and `evaluation`: their respective domain records and summaries.

Public type names and fields remain compatible. This release does not modify API JSON, Knowledge Backup, Client Storage, JSONL, migrations, or runtime behavior. Unit, E2E, and Mock Evaluation baselines remain unchanged. No Real API is invoked; V1.12.1 remains the deployment smoke baseline.
