# Enterprise Agent Hub V2.0.3 Release Notes

## Scope

V2.0.3 adds message actions for the latest completed turn without changing the V2.0.2 Conversation schema, NDJSON event protocol, context-window algorithm, Feedback API, or JSON Agent API.

## Highlights

- Copies only the plain visible Assistant answer, preserving line breaks while excluding metadata, Trace, sources, tools, Feedback, prompts, and rendered HTML.
- Regenerates only the latest completed Assistant answer through the existing streaming request path.
- Edits and resends only the latest paired user question with Enter, Shift+Enter, Escape, and IME-safe keyboard behavior.
- Uses one request controller for `send`, `retry`, `regenerate`, and `edit_resend`; the action is a safe scalar for local state, Ops, and Trace and never enters the prompt.
- Builds regenerate and edit-resend context from completed messages before the target turn, excluding both the old answer and the edited question's old form.
- Keeps the original stored turn and its Feedback during generation, failure, stop, or cancellation; only `answer_completed` can atomically replace it.
- Creates a new runId and fresh Feedback state after successful replacement. Old Feedback is never copied to the new answer.
- Recomputes an automatic title only when a successful edit changes the first user question. Manual titles remain unchanged.
- Extends the existing AbortController, request epoch, source conversation, target existence, completion fence, and message-id CAS protections.
- Keeps V2.0.2 stored conversations and the earlier Chat History migration compatible.

## Boundaries

- Historical Assistant answers may be copied, but only the latest completed answer can be regenerated.
- Only the latest paired user message can be edited.
- This release does not add arbitrary historical regeneration, editing of earlier turns, answer version retention, branch trees, server-side conversations, automatic summaries, or long-term memory.
- Automated tests use deterministic Mock streaming and do not call a Real API.

## Release Gate

- Unit tests: 28 files, 108/108 passed.
- Playwright E2E: 24/24 passed, including six message-action and revision scenarios.
- Full Mock Evaluation: 80/80, 100% pass rate.
- Unit tests, Playwright E2E, typecheck, Full Mock Evaluation, production build, production dependency audit, and `git diff --check` must complete before the stable tag is created.
- Full Mock Evaluation must remain 80/80 with a 100% pass rate.
- High or critical dependency findings block the release; known transitive moderate findings may remain documented when the suggested forced fix is incompatible.
