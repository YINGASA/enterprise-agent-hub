# V2.1.0 Release Notes — Conversation Context Manager

## 状态

V2.1.0 已完成本地开发与最终安全回归，但尚未 merge、tag、push 或部署。

## 新增能力

- **统一 Context Plan**：客户端仅提交受限的候选历史；服务端在 Router、RAG 与 Tool 执行后，统一完成 History Selection、Rolling Summary、Token Budget 和最终裁剪。
- **Token Budget**：模型窗口基线为 8192 tokens，保留 1200 输出 tokens，最大输入为 6800 tokens，包含 500 tokens 安全与序列化余量。超限时按 Tool、RAG、Selected History、Summary、最旧 Recent 的确定性顺序裁剪。
- **Smart Context Selection**：默认保留最近 4 个完整 turn；从更早历史中以关键词、数字/CJK 片段、Router scene/intent 和时间接近度确定性选择最多 2 个相关 turn。
- **Rolling Summary**：达到 8 个完整 turn 时，摘要较早的完整 turn 并保护最近 4 个。`throughMessageId` 指向最后一个已覆盖完整 turn 的 assistant 消息，后续仅增量处理新增 eligible turn。
- **Streaming Summary Patch**：`answer_completed` 可携带一次可选 Summary Patch；普通、Real Streaming、Real Fallback 和 Mock 都复用 Pipeline 已计算的同一 Patch。
- **Revision Safety**：Regenerate 与 Edit & Resend 使用有效历史前缀、CAS 和新的 runId；完成时消息变更与 Summary Patch 原子写入。Stop、失败、abort、会话切换和陈旧 epoch 不会持久化 Patch。
- **安全 Trace**：只包含策略、估算、预算、计数和枚举状态，不记录 Summary、历史正文、RAG 原文、Tool 原始输出、Prompt 或环境配置。

## 兼容性与边界

- V2.0.4 Conversation 与 Legacy Chat History 无需清空 localStorage 即可读取。
- localStorage key、Client Storage envelope version、Conversation 上限 10 个和单会话消息上限 100 条保持不变。
- 损坏 Summary 仅被丢弃，不会损坏 Conversation messages。
- 未增加第二次模型调用、数据库、对象存储、长期记忆、跨 Conversation 记忆、embedding 或用户画像。
- 未改变 Chat UI 布局，未升级 Node 或依赖，未进行云端部署。
