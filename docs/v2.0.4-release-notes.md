# Enterprise Agent Hub V2.0.4 Release Notes

## Product focus

V2.0.4 narrows the active product portfolio to enterprise knowledge-base questions, policy and material queries, business workflow automation, ecommerce service, RAG retrieval, business tools, and traceable evaluation. The former recruitment and career Agent is retired from the current product rather than hidden behind CSS or a feature flag.

## Scenario retirement

- Home, Chat recommendations, Scenarios, Tools, Knowledge, Evaluation, About, and current product documentation no longer present recruitment or career as an active capability.
- New Router and Mock runs cannot select the retired scenario, intent, knowledge pack, or tool. Direct career questions fall back to general handling without claiming a specialist workflow.
- The 80-case Full Mock suite replaces all 16 retired cases with cases for implemented enterprise policy, reimbursement, approval, ecommerce, RAG, clarification, tools, context, and fallback behavior. The quality threshold is unchanged.
- No new Knowledge Pack is introduced in this release.

## Compatibility and data safety

- Legacy scenario, intent, tool, and pack identifiers remain parseable only where old Conversation, Chat History, Evaluation, source, or Ops records require them. Current UI labels those values as historical or retired.
- Existing messages, runId values, Feedback records, storage keys, backup files, and schemas are not rewritten or deleted.
- Built-in recruitment demonstration knowledge is removed from the active default library. User-uploaded or pasted documents are never deleted or filtered because of their subject, and their enabled state and RAG participation remain unchanged.
- V2.0.3 multi-turn context, streaming, stop/retry, answer copy, regenerate, and edit-resend contracts remain unchanged.

## Validation boundary

- Local automation uses Mock mode and does not call a Real API.
- Unit tests, Playwright E2E, typecheck, Full Mock Evaluation, production build, dependency audit, and `git diff --check` are release gates.
- Full Mock Evaluation must remain 80/80 with a 100% pass rate. High or critical dependency findings block release; documented transitive moderate findings may remain when the forced fix is incompatible.
- The completed local gate contains 29 unit-test files with 116/116 tests and 28/28 Playwright E2E scenarios.
