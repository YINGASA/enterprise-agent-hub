# V2.0.1 Release Notes

V2.0.1 redesigns the Chat Workspace without changing the V2.0.0 multi-turn context API, storage key, or message-role contract.

## Chat workspace

- Desktop uses a fixed-width conversation sidebar and a flexible chat pane. Mobile uses an accessible conversation drawer with Escape, backdrop, close-button, and selection handling.
- Messages form one chronological stream: user bubbles align right, Assistant answers align left, and the Composer remains visible at the bottom while only the message area scrolls.
- The sidebar supports local title search, deterministic first-question titles, inline rename, current selection, delete confirmation, and updated-time ordering.
- Empty new conversations stay in memory and are materialized only after the first successful response. Repeated “new conversation” actions do not create duplicate persistent drafts.
- Long conversations follow the latest message only while the user remains near the bottom. Scrolling upward reveals a “back to latest” action without forcing the viewport down.

## Message ownership and safety

- Context counts, response mode, source references, tool status, safe Trace summaries, and Feedback controls are rendered under the Assistant message that produced them.
- Conversation storage keeps only whitelisted source metadata, tool names/statuses, and step names/statuses/durations. It does not store source or chunk bodies, tool inputs/results, full Trace input/output, prompts, or Feedback content.
- Pending user messages and Assistant placeholders appear immediately but are committed only after success. Failure keeps the draft available for retry without duplicating the user message.
- Requests are tied to their origin conversation through a request epoch, explicit conversation ID, and AbortController. Switching or deleting a conversation prevents a late response from writing into another conversation or recreating a deleted one.

## Compatibility and scope

Existing V2.0.0 conversations receive idempotent local titles, and V1.12.5 Chat History migration remains available. Feedback, Evaluation, RAG Test History, the Agent API payload, and context budgets are unchanged. This release does not add streaming output, stop/regenerate actions, server-side conversation storage, automatic summaries, or long-term memory. Real API was not called locally; controlled smoke testing remains a deployment step.

Release validation covers 76 unit tests, 16 Playwright E2E scenarios, Full Mock Evaluation 80/80, typecheck, production build, and the production dependency audit.
